WWW: It has also commonly become known simply as the Web and is an information system of interlinked hypertext documents that are accessed via the Internet.

HTTP: Short for HyperText Transfer Protocol. Is the underlying protocol used by the Web and defines how messages, between Servers and browsers, are formatted and transmitted.

HTLM: Short for HyperText Markup Language. A markup language is a set of markup tags, so HTML is a language for describing web documents (web pages) where those are described by HTML tags.

Browser: The purpose of a browser is to read HTML documents and display them. Be aware that the browser does not display the HTML tags, but uses them to determine how to display the document.

Even though it seems pure magic, what is really behind is just a formal procedure to establish a straightforward communication between your computer and a web server. As in the ancient times when an embassador went to another kingdom to meet the king, an elaborate protocol where established to keep the relations between both nations in proper manner. Something similar happens when you try to access a web page throughout the internet. A specific protocol is followed by the computers to get and fetch the page that you are requesting.

The actors, by order of appearance are: the User, the Internet and the web Servers.

The story starts when you, after some time of hard work finally get a chance for an interview for your ideal job. The interview will be in your home and you are planning a formal dinner to close the deal but you forgot that you need to finish a report for your boss and don't have much time to set everything for the dinner to make the good impression you want, so now you has to figure it out how resolve the mess.

Almost at the point you're in panic, remember that your bro (dont be confuse, not your brother, your browser), is available and he can help you setting the table for you. But there is a little problem, he doesn't know much about politeness and you can't afford the minimum error, everything has to be perfect!

First, you thought write a precisely instruction manual for your bro but you remember he is not a good reader too - at this point you are thinking your bro is stupid - the only thing he is good enough is solving puzzle. There is when you got the idea about HTML (Help me with This Mess pLease).

A simple way to tell your bro what he has to do to set the table. Using small pieces of information, such as tags, to name everything you needs on the table, for example you use <c> for candle, <w> for wine, <f> for fork, etc.

To avoid creates a bunch of tags, one for red wine or white candle, you use attributes to specify that kind of information, for example <t type='round'> for round table. At the end you has already created a nice and smooth system to tell your bro how he has to set the table, moreover you realize that the system can be used to set any kind of table for any kind of dinner.

By the way, when you arrive everything is done, the interview was perfect and got the job. But when you thought to share with the world about your nice system, you realize that someone else has already created HTML and use a showy name for that, "HyperText Markup Language" ....Ha!

Three kind of files conforms to a Web Document: HTML, CSS and Job Descriptors.

HTML is the structure of the document, something similar to the rooms in a house.

CSS is the style of the document, such as the furniture or the color of the walls in the house.

Job Descritors are the interacting components in the document, such as the remote control for TV or open the garage.

HTML documents are described by HTML tags which describe different document content. HTML tags are keywords surrounded by angle brackets. Normally the Tags come in pairs like <p> and </p>, where the first tag in a pair is the start tag (often called the opening tag), the second tag is the end tag (often called the closing tag). The purpose of the end tag is "Close" the content between both tags and tells the browser where the format of the content ends. The end tag is written like the start tag, but with a slash before the tag name.

The HTML element is everything from the start tag to the end tag, namely are written with a start tag, with an end tag, with the content in between: <tagname>content</tagname>. Attributes provide additional information about HTML elements and are always specified in the start tag throughtout name/value pairs like: name="value".

The W3C Document Object Model (DOM) is a platform and language-neutral interface that allows programs and scripts to dynamically access and update the content, structure, and style of a document. The HTML DOM defines the objects and properties of all document elements, and the methods (interface) to access them.

The DOM views an HTML document as a tree-structure. All nodes can be accessed through the tree. Their contents can be modified or deleted, and new elements can be created. The node tree shows the set of nodes, and the connections between them. The tree starts at the root node and branches out to the text nodes at the lowest level of the tree.

CSS stands for Cascading Style Sheets and defines how HTML elements are to be displayed.

CSS solves a big problem due to HTML was NEVER intended to contain tags for formatting a document. HTML was intended to define the content of a document.

When tags like <font>, and color attributes were added to the HTML 3.2 specification, it started a nightmare for web developers. Development of large web sites, where fonts and color information were added to every single page, became a long and expensive process. To solve this problem, the World Wide Web Consortium (W3C) created CSS.

CSS selectors allow you to select and manipulate HTML elements. CSS selectors are used to "find" (or select) HTML elements based on their id, class, type, attribute, and more. The three most used kind of CSS selectors are 1) element selectors, 2) id selectors and 3) class selectors.

The element selector selects elements based on the element name. You can select all <p>, elements on a page.

The id selector uses the id attribute of an HTML element to select a specific element. To select an element with a specific id, write a hash character, followed by the id of the element.

The class selector selects elements with a specific class attribute. To select elements with a specific class, write a period character, followed by the name of the class.

All HTML elements can be considered as boxes. In CSS, the term "box model" is used when talking about design and layout.

The CSS box model is essentially a box that wraps around HTML elements, and it consists of: margins, borders, padding, and the actual content. The box model allows us to add a border around elements, and to define space between elements.

The Box Model is composed of:

Content - The content of the box, where text and images appear. Padding - Clears an area around the content. The padding is transparent. Border - A border that goes around the padding and content. Margin - Clears an area outside the border. The margin is transparent.
